# 상태관리 규칙

## 개요
이 문서는 앱 상태관리 방법과 규칙을 정의합니다.

## 상태 분류
- **전역 상태**: 인증, 사용자 정보, 앱 설정 → Context API 또는 Zustand
- **서버 상태**: API 데이터 캐싱 및 동기화 → React Query (TanStack Query)
- **로컬 상태**: 단일 컴포넌트 UI 상태 → useState

## 전역 상태 규칙
- Context는 src/contexts/ 에 위치
- 하나의 Context는 하나의 관심사만 담당
- Provider는 App.tsx에서 중첩하여 등록

## 서버 상태 규칙
- 쿼리 키는 src/queryKeys.ts에 중앙 관리
- 캐시 시간: 기본 5분, 중요 데이터 1분
- 낙관적 업데이트는 UX에 중요한 작업에만 적용

## 주의사항
- 전역 상태에 파생 데이터를 저장하지 않는다 (계산해서 사용)
- 불필요한 리렌더링을 방지하기 위해 상태를 최대한 아래로 내린다
